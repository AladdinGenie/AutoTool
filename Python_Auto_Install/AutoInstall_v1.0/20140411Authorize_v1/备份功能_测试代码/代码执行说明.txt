数据库备份执行代码说明：
1、修改config.txt文件的参数：
	set_sleeptime：执行代码需要等待的时间（建议不修改）
	set_sysday：运行脚本的次数（建议不小于15）
	set_backupday：程序设计如此（不能修改），值为15
2、testBackup.py文件的参数：
	localfile：是运行脚本后存放的日志文件位置（可自行修改）
3、verify.py文件参数：
	localfile：意义同上（testBackup.py和verify.py中的localfile的值要相同）
	path：备份存放的位置/sunshine/backups，一般不修改
4、首先将3个文件放入服务器端，且3个文件放在同一目录下。
5、在服务器上运行代码，命令如下：
	python testBackup.py
6、验证点：
	进入localfile目录下，查看里面的日志文件的最后，看到一个由'[]'包含的目录，里面有15个以','隔开的数字串
	或者此时直接使用CRT连接到服务器，进入备份文件存放的位置，查看里面的文件是否保留15个，若再次运行脚本时，
	它删除的是日期早的文件（或数字小的文件）；
	查看平台的操作日志界面，它显示的日期是15天的内容（即删除15天前的日志记录）
